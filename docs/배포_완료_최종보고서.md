## 2. 프론트엔드 배포 (✅ 완료)

### 배포 URL
- **메인**: https://lid-publishing-helper.web.app
- **대체**: https://lid-publishing-helper.firebaseapp.com

### 빌드 문제 해결

#### 문제
Vite v5.4.11에서 `esbuild` minifier 사용 시 빌드가 "rendering chunks" 단계에서 멈춤

#### 원인
- Vite v5.4.11의 알려진 버그
- esbuild minifier와의 호환성 문제

#### 해결 방법
`vite.config.js`에서 minify 옵션 변경:
```javascript
build: {
  sourcemap: false,
  minify: 'terser',  // esbuild → terser로 변경
  target: 'esnext'
}
```

#### 결과
- ✅ 빌드 성공 (7.07초)
- ✅ 파일 크기 최적화
  - 최초 빌드 (minify: false): 2,740KB
  - 최종 빌드 (minify: terser): 1,174KB
  - 압축률: 57% 감소

## 4. 작동 흐름

```
사용자
  ↓ 글쓰기 페이지 접속
  ↓ Step 3: 근거자료 업로드
  ├─ PDF 파일
  ├─ TXT 파일
  ├─ Excel/CSV 파일
  └─ 이미지 파일 (최대 3개, 10MB)
  ↓
geminiService.js (프론트엔드)
  ↓ 파일 타입별 처리
  ├─ TXT: 브라우저에서 직접 읽기
  ├─ PDF: extractPdfContent (Cloud Functions)
  ├─ Excel: extractExcelContent (Cloud Functions) ← 신규
  └─ 이미지: extractImageContent (Cloud Functions) ← 신규
  ↓
generateArticle (Cloud Functions)
  ↓ 1. Firestore에서 AI 지침 불러오기
  ↓ 2. 태그 치환
  │    [분량지침] → "A4 10~12쪽 분량으로"
  │    [사용자주제] → 실제 주제
  │    [근거자료내용] → 추출된 파일 내용 ← 신규
  ↓ 3. Gemini 2.5 Pro API 호출
  ↓
생성된 글 (근거자료 반영됨) → 사용자
```
