## 작동 흐름 (최종)

```
1. 관리자 페이지
   ├─ AI 지침 수정 → Firestore 저장 (settings/ai_prompts)
   └─ bracket placeholder 형식: [분량지침], [근거자료내용] 등

2. 사용자가 글쓰기 (Step 3 - 선택사항)
   ├─ 주제, 사례 입력 (Step 1-2)
   ├─ PDF 업로드 (법령/지침)
   ├─ 근거자료 업로드 (통계, 뉴스 등) ← 신규!
   │  - 최대 3개 파일
   │  - 최대 10MB
   │  - 형식: PDF, TXT, Excel, CSV, 이미지
   ├─ 구조적 문제점, 대안 입력 (선택)
   └─ "글 생성하기" 클릭

3. geminiService.js (프론트엔드)
   ├─ PDF 내용 추출 (extractPdfContent)
   ├─ 근거자료 내용 추출 ← 신규!
   │  ├─ TXT: extractTextContent() - 직접 읽기
   │  ├─ PDF: extractPdfContent() - Cloud Functions
   │  ├─ Excel/CSV: extractExcelContent() - Cloud Functions
   │  └─ 이미지: extractImageContent() - Cloud Functions (Vision API)
   └─ Cloud Functions로 전송
      - pdfContents
      - referenceContents ← 신규!

4. Cloud Functions (generateArticle)
   ├─ Firestore에서 최신 AI 지침 불러오기 ← 신규!
   ├─ 분량 지침 생성 (A4 10~12쪽 등)
   ├─ 태그 치환 ← 신규!
   │  ├─ [분량지침] → "A4 10~12쪽 분량으로"
   │  ├─ [사용자주제] → 사용자 입력
   │  ├─ [사용자사례] → 사용자 입력
   │  ├─ [PDF내용] → PDF 추출 내용
   │  ├─ [근거자료내용] → 근거자료 추출 내용 ← 신규!
   │  ├─ [구조적문제점] → 사용자 입력 or AI 생성
   │  └─ [대안제시] → 사용자 입력 or AI 생성
   ├─ Gemini 2.5 Pro API 호출
   └─ 생성된 글 반환 (JSON 형식)

5. 사용자
   └─ 근거자료가 반영된 글 확인 ✅
```

## 주요 기술 구현

### Bracket Placeholder 시스템
```javascript
// 관리자가 수정 가능한 프롬프트 템플릿
const template = `...
주제: [사용자주제]
근거자료: [근거자료내용]
...`

// Cloud Functions에서 치환
const finalPrompt = template
  .replace('[사용자주제]', customTopic)
  .replace('[근거자료내용]', referenceContents)
```

### 파일 타입별 처리
- **TXT**: FileReader로 직접 읽기 (클라이언트)
- **PDF**: Gemini PDF API (Cloud Functions)
- **Excel/CSV**: xlsx 라이브러리 (Cloud Functions)
- **이미지**: Gemini Vision API (Cloud Functions)

### Firestore 동적 프롬프트 로딩
```javascript
async function getAiPrompt() {
  const docRef = db.collection('settings').doc('ai_prompts')
  const docSnap = await docRef.get()
  return docSnap.data().articleGeneration || DEFAULT_ARTICLE_PROMPT
}
```
