## 2. Claude Code에서 빌드/배포할 때

Git Bash(MSYS2) 환경에서 `vite build`가 렌더링 단계에서 멈추는 문제가 있음.
**이건 프로젝트 문제가 아니라 Git Bash + Node v24 + pipe stdout 조합의 환경 문제.**

아이다(SEL HUB) 등 다른 프로젝트도 Claude Code에서 동일하게 발생할 수 있음.
사람이 직접 PowerShell/CMD에서 실행하면 문제 없음.

### 우회 방법
```bash
# 빌드
cmd.exe //c "cd /d C:\PRODUCT\LID && npx vite build"

# 배포
cmd.exe //c "cd /d C:\PRODUCT\LID && npx firebase deploy --only hosting"
```

---

## 3. 문제 해결

| 증상 | 해결 |
|------|------|
| 빌드 멈춤 (Claude Code) | `cmd.exe //c "npx vite build"` |
| 빌드 멈춤 (직접 실행) | PowerShell/CMD에서 실행 |
| Functions 배포 실패 | `firebase functions:log` 로 로그 확인 |
| 권한 오류 | `firebase login --reauth` |
| 롤백 필요 | Firebase Console > Hosting > 릴리스 기록 > 이전 버전 복원 |

## 4. Vite 빌드 크래시 문제 기록 (2026-03-09)

### 증상
- `vite build` 실행 시 `✓ 1796 modules transformed.` 출력 후 프로세스 크래시
- Windows 종료 코드: `3221226505` (= `0xC0000409` = `STATUS_STACK_BUFFER_OVERRUN`)
- 에러 메시지 없이 조용히 죽어서 hang처럼 보임

### 근본 원인
**`dist/` 또는 `node_modules/.vite/` 캐시 오염.**
이전 빌드 시도(실패한 빌드, esbuild 빌드 등)로 인해 dist 폴더와 Vite 캐시가 불일치/오염 상태가 되어 rollup 렌더링 단계에서 크래시 발생.

### 해결 방법
```bash
rm -rf dist node_modules/.vite
npm run build
```

### 검증 과정
- 같은 rollup 바이너리(4.55.1), 같은 vite(5.4.21)로 SEL HUB는 8448 모듈 빌드 성공
- LID에서도 `dist/`와 `node_modules/.vite/` 삭제 후 깨끗한 상태에서 빌드하면 5.72초 만에 정상 완료
- 다른 디렉토리에 동일 소스 복사해도 정상 빌드 확인

### 빌드 실패 시 체크리스트
1. `rm -rf dist node_modules/.vite` 후 재빌드
2. 그래도 안 되면 `rm -rf node_modules package-lock.json && npm install` 후 재빌드
3. `package.json`에 `"overrides": { "rollup": "4.55.1" }` 추가하여 안정 버전 고정 권장
