## 1. 배경 및 목적

### 현재 상태
- **프로젝트**: LID (제도혁신분과 출판 도우미)
- **위치**: `c:/PRODUCT/LID/`
- **배포**: Firebase Hosting (`lid-publishing-helper.web.app`)
- **Git 상태**: `git init`만 된 상태, 원격 저장소 없음, 커밋 이력 0건
- **사용**: 분과원 전용 비공개 서비스

### 목적
- 다른 분과 교사들이 **자기만의 홈페이지**를 직접 만들 수 있도록
- 현재 프로젝트를 **템플릿**으로 GitHub에 공개
- 각자 clone → 자기 Firebase 연결 → 독립 운영

### 전략
- **원본(`c:/PRODUCT/LID/`)은 절대 건드리지 않음**
- `c:/PRODUCT/template/`에 복사본 생성 → 민감정보 제거 → GitHub 푸시

## 5. 주의사항

### 원본 보호
- `c:/PRODUCT/LID/`는 **어떤 파일도 수정/삭제하지 않음**
- 복사 작업만 수행, 원본은 그대로 실서비스 유지

### 다른 교사들이 사용하려면 각자 준비할 것
1. **Firebase 프로젝트** 생성 (무료 Spark 플랜 가능)
2. **Firestore Database** 활성화
3. **Firebase Storage** 활성화
4. **Firebase Hosting** 설정
5. **(선택) Google AI Studio**에서 Gemini API 키 발급 (AI 글쓰기 기능용)
6. **(선택) Firebase Cloud Functions** - Blaze 플랜 필요 (유료, 하지만 무료 범위 넉넉)

### Firebase 웹 API 키 공개 관련
- Firebase 웹 apiKey는 사실 "공개 식별자" 성격이라 노출 자체가 치명적이진 않음
- 하지만 **Firestore 보안 규칙**이 핵심 → 템플릿에 보안 규칙 가이드 필수
- 현재 LID의 보안 규칙도 확인 후 템플릿에 포함하면 좋음

### Git에 올려도 서비스 작동에 영향 없음
- Git은 소스코드 버전관리일 뿐, Firebase 배포와 독립적
- 현재 LID 서비스는 Firebase에 이미 배포된 상태이므로 영향 없음
