## 0. 사용자가 진짜 원하는 것 (Deep Intent Analysis)

### 표면적 요청
"파트별 진도를 시각화해주세요"

### 진짜 원하는 것

현재 페이지는 16개 박스에 이름과 파일이 들어가는 것이 전부다.
올리면 끝, 개인의 행동으로 마무리된다.

사용자가 원하는 것은 팀 전체의 공동 출판물이 만들어지는 과정을
한 눈에 보여주는 '살아있는 공동 작전상황판'이다.

| 핵심 필요 | 설명 |
|------|------|
| 집단 책임감 | "내가 아직 안 올렸구나"를 스스로 보게 됨 |
| 집단 성취감 | "Part 1 완성됐다!" 의 기쁨 공유 |
| 출판 준비도 확인 | 지금 어디까지 왔는가 한눈에 파악 |
| 유연한 운영 | 담당자 바뀌어도 관리자가 쉽게 수정 |
| 낮은 업로드 장벽 | 여전히 클릭 한 번에 올릴 수 있어야 함 |

## 1. 현황 분석

### 현재 구조 (문제점)

```
[ 이창건 ] [ 이영은 ] [ 김미진 ] [ 신주현 ] ...
[ 안형구 ] [ 공정욱 ] [ 장미선 ] [ 김수진 ] ...
...
```

- 파트 개념 없음. 주제 개념 없음.
- 파일 올리면 박스 색만 바뀜. 진도 없음.
- 출판 준비 상태를 알 방법 없음.
- 누가 어떤 주제를 맡았는지 보이지 않음.

### 2026년 연간 발제 일정 기반 데이터

| 파트 | 발제 횟수 | 주제 수 |
|------|----------|-------|
| Part 1 - 학교라는 시스템의 기초 | 6회 | 12개 |
| Part 2 - 무엇을 배우고 어떻게 평가하나 | 5회 | 11개 |
| Part 3 - 갈등 해결과 소통의 현장 | 5회 | 8개 |
| 합계 | 16회 | 31개 |

## 3. 핵심 기능 사양

### 메인 페이지 (글 파일로 올리기)

1. 전체 진도 요약 헤더 - 전체 몇 % 완성
2. 파트별 카드 - 각 파트의 진도바 + 완성 표시
3. 주제 목록 - 각 파트 카드 안에 주제별 행 (담당자, 상태, 업로드)
4. 완성 시 표시 - 100%가 되면 파트 카드에 완성 표시

### 관리자 페이지 추가 기능

기존: 카테고리(파트) 이름 변경
추가:
- 주제 추가 / 삭제 / 이름 변경
- 주제별 담당자 변경
- 발제 날짜 변경
