## ⏳ 프론트엔드 배포 대기 중

프론트엔드 빌드가 자동으로 완료되지 않았습니다.
다음 방법 중 하나로 직접 배포해주세요.

### 방법 1: 명령 프롬프트 사용 (권장)

새 명령 프롬프트 또는 PowerShell을 열고:

```bash
cd c:\PRODUCT\LID\frontend
npm run build
```

빌드가 완료되면 (약 1-2분 소요):

```bash
cd c:\PRODUCT\LID
firebase deploy --only hosting
```

### 방법 2: 배치 파일 사용

1. 파일 탐색기에서 `c:\PRODUCT\LID` 폴더 열기
2. `build-and-deploy-hosting.bat` 파일 생성 (아래 내용 복사)
3. 파일 더블클릭하여 실행

**build-and-deploy-hosting.bat 내용:**
```batch
@echo off
echo ================================
echo Frontend Build and Deploy
echo ================================
echo.

cd c:\PRODUCT\LID\frontend
echo [1/2] Building frontend...
call npm run build

if %ERRORLEVEL% NEQ 0 (
    echo.
    echo [ERROR] Build failed!
    pause
    exit /b %ERRORLEVEL%
)

echo.
echo [SUCCESS] Build completed!
echo.
echo [2/2] Deploying to Firebase Hosting...
cd ..
call firebase deploy --only hosting

if %ERRORLEVEL% NEQ 0 (
    echo.
    echo [ERROR] Deployment failed!
    pause
    exit /b %ERRORLEVEL%
)

echo.
echo ================================
echo Deployment Successful!
echo ================================
pause
```

### 방법 3: 이미 빌드된 경우 (dist 폴더가 있다면)

`c:\PRODUCT\LID\frontend\dist` 폴더가 이미 존재하는 경우:

```bash
cd c:\PRODUCT\LID
firebase deploy --only hosting
```

#### 테스트 1: AI 지침 수정
1. 관리자 페이지 접속
2. AI 글쓰기 지침 수정 버튼 클릭
3. 내용 수정 후 저장
4. Firestore Console에서 `settings/ai_prompts` 문서 확인

#### 테스트 2: TXT 파일 업로드
1. 간단한 TXT 파일 준비 (예: "2024년 교육부 통계: 학생 수 100명")
2. 글쓰기 Step 3에서 TXT 파일 업로드
3. 글 생성

#### 테스트 3: Excel 파일 업로드

4. Chrome DevTools Network 탭에서 extractExcelContent 호출 확인
5. 생성된 글에 Excel 데이터 반영 확인

#### 테스트 4: 이미지 업로드 (OCR)

4. Chrome DevTools Network 탭에서 extractImageContent 호출 확인
5. 생성된 글에 이미지 내용 반영 확인
