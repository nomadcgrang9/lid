# 260309 캘린더 일정 연동 구현 계획

## 1. 현재 상태 분석

### 현재 LandingPage 구조
- 중앙 정렬 단일 컬럼 레이아웃 (max-width: 520px)
- 구성: 이미지 → 소개 텍스트 → 버튼 3개 (회비계좌, 줌 미팅주소, 후원하기)
- 깔끔하고 심플한 구조

### 일정 데이터 모델 (Firestore `schedules` 컬렉션)
```
{
  year: number,          // 2026
  round: number,         // 회차 (1, 2, 3...)
  date: string,          // "4월3일", "3월15일" (텍스트 형식)
  topic: string,         // 주제 및 내용
  presenter: string,     // 진행/발표자
  recorder: string,      // 기록자
  meetingNote: string,   // 모임 기록
  attendees: string      // 참석자
}
```

### 핵심 과제
- `date` 필드가 "4월3일" 같은 **텍스트 형식** → 캘린더 날짜와 매핑하려면 파싱 로직 필요
- `year` + `date` 조합으로 실제 Date 객체 생성 가능 (예: 2026 + "4월3일" → 2026-04-03)

### 디자인 시스템
- Primary: `#3B82F6` (블루)
- Primary Dark: `#1E40AF`
- Surface: `#f8fafc`
- Border: `#e2e8f0`
- Text: `#1e293b` / Secondary: `#64748b`
- Font: Noto Sans KR
- 라운드: 8px~16px

## 2. 구현 범위

### 추가할 것
- LandingPage에 **미니 캘린더 위젯** 추가
- 캘린더에 일정(schedules) 데이터를 dot/badge로 표시
- 날짜 클릭 시 해당 일정 요약 표시
- "전체 일정 보기" 링크 → SchedulePage로 이동

### 변경하지 않을 것
- SchedulePage 자체는 변경 없음
- 일정 데이터 구조 변경 없음
- 기존 사이드바, 라우팅 구조 변경 없음

### 기술적 고려사항
- 외부 라이브러리 없이 **순수 React로 커스텀 캘린더** 구현 (사이트 컬러 일관성 유지 용이)
- `date` 텍스트 → Date 파싱 유틸리티 함수 필요
- 캘린더는 월 단위 네비게이션 (< 이전 / 다음 >)

## 5. 공통 기술 구현 사항 (어떤 목업이든)

### 5-1. 날짜 파싱 유틸리티
```
"4월3일" + year:2026 → new Date(2026, 3, 3)  // month는 0-indexed
"12월25일" + year:2026 → new Date(2026, 11, 25)
```
- 정규식으로 `(\d+)월(\d+)일` 패턴 추출

### 5-2. 캘린더 컴포넌트 구조
```
CalendarWidget
├── 헤더 (< 2026년 3월 >)
├── 요일 행 (일 월 화 수 목 금 토)
├── 날짜 그리드 (6주 × 7일)
│   ├── 빈 셀 (이전/다음 달)
│   ├── 일반 날짜
│   ├── 오늘 날짜 (원형 파란 배경)
│   └── 일정 있는 날짜 (하단 작은 점)
└── 선택된 날짜의 일정 프리뷰
```

### 5-3. 데이터 흐름
```
LandingPage mount
  → getAllSchedules() 또는 getSchedules(currentYear) 호출
  → 날짜 파싱하여 Map<dateString, schedule[]> 생성
  → CalendarWidget에 전달
  → 날짜 클릭 시 해당 일정 하이라이트
```

### 5-4. 스타일 일관성
- 캘린더 배경: `var(--background-color)` (white)
- 캘린더 카드 border: `1px solid var(--border-color)`
- 오늘 날짜: `var(--primary-color)` 배경 + white 텍스트
- 일정 dot: `var(--primary-color)` 작은 원
- 요일 헤더: `var(--text-secondary)` 색상
- 토요일: `var(--primary-color)`, 일요일: `var(--error-color)`
- hover: `var(--surface-color)` 배경
- 선택된 날짜: `var(--primary-color)` 테두리

## 6. 추천

**개인적 추천: 목업 C (카드형 2단)**

이유:
1. 참고 사이트(스샷1)의 2단 레이아웃 느낌을 잘 살림
2. 카드로 감싸서 시각적 구분이 깔끔
3. 기존 소개 콘텐츠가 카드 안에서 자연스럽게 유지됨
